# 07 — Shadow Runtime Source Generation Spec

## Purpose

Evidence expects source queries under `sources/[source_name]/`. The content workspace should remain clean, so generated source files must live in the shadow runtime.

## Generated source location

```text
shadow-runtime/sources/files/
  connection.yaml
  orders.sql
  customers.sql
```

Do not generate this directory under the user-facing content workspace.

## Source name

MVP source name:

```text
files
```

Registered tables become:

```text
files.<alias>
```

## Generated `connection.yaml`

The generated connection file must not contain secrets.

Example:

```yaml
# Generated by cmux-evidence data refresh. Do not edit by hand.
name: files
type: duckdb
options:
  filename: ":memory:"
```

If Evidence requires a different exact DuckDB connection shape, follow current Evidence/DuckDB conventions in the repo.

## Generated SQL files

### CSV

```sql
-- Generated by cmux-evidence data refresh. Do not edit by hand.
-- Source file: data/orders.csv
select *
from read_csv_auto('workspace-data/orders.csv')
```

### TSV

```sql
-- Generated by cmux-evidence data refresh. Do not edit by hand.
-- Source file: data/orders.tsv
select *
from read_csv_auto('workspace-data/orders.tsv', delim='\t')
```

### Parquet

```sql
-- Generated by cmux-evidence data refresh. Do not edit by hand.
-- Source file: data/orders.parquet
select *
from read_parquet('workspace-data/orders.parquet')
```

### JSON / JSONL

```sql
-- Generated by cmux-evidence data refresh. Do not edit by hand.
-- Source file: data/events.jsonl
select *
from read_json_auto('workspace-data/events.jsonl')
```

## Path rule

Generated SQL should reference the shadow runtime's `workspace-data` link/copy, not arbitrary absolute paths.

Preferred:

```sql
read_csv_auto('workspace-data/orders.csv')
```

Avoid unless absolutely necessary:

```sql
read_csv_auto('/absolute/path/to/workspace/data/orders.csv')
```

## Regeneration rule

On every refresh:

1. Ensure `sources/files/` exists.
2. Rewrite `connection.yaml`.
3. Remove stale generated `.sql` files for aliases no longer ready.
4. Write `.sql` files for `ready` tables.
5. Leave non-generated files alone only if they are outside `sources/files/`.

Simpler MVP: treat everything under `sources/files/` as generated and rewrite it completely.

## Empty workspace behavior

If no files are registered:

- `sources/files/` may be absent or contain only generated `connection.yaml`.
- dev server should not fail only because no data exists.
- print a helpful message.

## Evidence source execution

When files exist:

```bash
npm run sources -- --sources files
```

When no files exist, skip this command.

## Safety

Generated SQL must never reference:

- `.env*`,
- parent paths using `..`,
- absolute paths outside workspace/runtime roots,
- secret files,
- unsupported external URLs.
